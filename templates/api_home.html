📚 Comment utiliser l'interface Swagger

🎯 Swagger UI est similaire à Django REST Framework !
Vous pouvez tester tous les endpoints directement depuis votre navigateur.

Étapes pour tester l'API :

  1. Ouvrez Swagger UI en cliquant sur le bouton ci-dessus
  2. Testez l'endpoint de connexion :
    • Cliquez sur POST /api/v2/login
    • Cliquez sur "Try it out"
    • Utilisez vos identifiants : 
      {
  "email": "votre.email@example.com",
  "password": "votre_mot_de_passe"
}
    • Cliquez sur "Execute"
    • Copiez le access_token reçu
  3. Autorisez-vous :
    • Cliquez sur le bouton vert "Authorize" en haut à droite
    • Entrez : Bearer <votre_token>
    • Cliquez sur "Authorize" puis "Close"
  4. Testez les autres endpoints qui nécessitent l'authentification 🎉

🔑 Endpoints principaux (API v2)

POST /api/v2/register
Créer un nouveau compte utilisateur
POST /api/v2/login
Se connecter et obtenir un token JWT
GET /api/v2/user/profile
Obtenir le profil de l'utilisateur connecté 🔒
GET /api/v2/user/stats
Obtenir les statistiques d'enregistrement 🔒
GET /api/v2/languages
Liste des langues disponibles 🔒
GET /api/v2/sentences/next
Obtenir la prochaine phrase à enregistrer 🔒
POST /api/v2/recordings
Sauvegarder un enregistrement audio 🔒
GET /api/v2/recordings
Liste des enregistrements de l'utilisateur 🔒
GET /api/v2/provinces
Liste des provinces de la RDC (public)
🔒 Authentification requise : Les endpoints marqués 🔒 nécessitent un token JWT dans le header Authorization.

💡 Informations importantes

  • Base URL v1 : /api/
  • Base URL v2 (documentée) : /api/v2/
  • Documentation interactive : /api-docs/
  • Format : JSON
  • Authentification : JWT Bearer Token (expire après 30 jours)
  • Taille max fichiers : 50 MB
