# Contributing to Lugayetu

Thank you for your interest in contributing to **Lugayetu**.

Lugayetu is a research and technology project aimed at **preserving Congolese languages and developing artificial intelligence tools for low-resource languages**. Contributions from developers, researchers, linguists, and native speakers are welcome.

This document explains how you can contribute to the project.

---

# Types of Contributions

There are several ways to contribute to Lugayetu.

## 1. Technical Contributions

Developers can contribute by improving the platform or adding new features.

Examples include:

- fixing bugs
- improving performance
- improving the user interface
- adding new APIs
- improving documentation
- improving data processing scripts
- developing tools for dataset preparation

---

## 2. Linguistic Contributions

Lugayetu also welcomes contributions from linguists and native speakers.

Possible contributions include:

- adding new sentences in local languages
- correcting translations
- improving linguistic quality
- suggesting new languages to support
- validating linguistic datasets

These contributions help build **high-quality datasets for Natural Language Processing (NLP)**.

---

## 3. Voice Data Contributions

Native speakers can contribute by recording **audio samples of sentences**.

These recordings are used to build:

- speech datasets
- speech recognition systems
- speech synthesis models
- future speech-to-speech translation systems

Voice contributions are essential for developing **AI speech technologies for Congolese languages**.

---

# How to Contribute (Developers)

## 1. Fork the repository

Create your own copy of the project on GitHub.

## 2. Clone the repository

Download the repository to your local machine.

## 3. Create a new branch

Create a branch for your modification or feature.

Example:

feature-new-dataset-tools

or

fix-audio-upload

## 4. Make your changes

Implement your feature or fix the issue.

Please try to:

- write clear and readable code
- document your changes
- keep changes focused and minimal

## 5. Submit a Pull Request

Once your changes are ready, open a **Pull Request** describing:

- what you changed
- why the change is useful
- any related issue

The maintainers will review your contribution.

---

# Reporting Issues

If you find a bug or want to suggest an improvement, please open an **issue** on the repository.

When reporting a bug, try to include:

- a clear description of the problem
- steps to reproduce the issue
- screenshots or logs if possible

---

# Data and Ethical Considerations

Lugayetu collects **linguistic and voice data from native speakers**.  
Contributors must respect ethical guidelines when working with this data.

Please ensure that:

- contributors understand how their data will be used
- the data is not used to harm communities
- linguistic resources are used respectfully

---

# Research Contributions

Researchers working in the following areas are encouraged to collaborate:

- Natural Language Processing (NLP)
- Neural Machine Translation
- Speech Recognition
- Speech Synthesis
- Low-resource language technologies

Lugayetu aims to become a **research platform for AI applied to Congolese languages**.

---

# Code of Conduct

All contributors are expected to behave respectfully and professionally.

Discrimination, harassment, or disrespectful behavior will not be tolerated.

---

# Acknowledgment

All meaningful contributions to Lugayetu may be acknowledged in:

- project documentation
- research publications
- future dataset releases
